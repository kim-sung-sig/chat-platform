# pre-feature-check

## 목적
- 새 작업을 시작하기 전에 기존 스킬과 외부 스킬을 비교한다.
- 유용한 스킬을 발굴하여 "맞춤형" 역량을 점진적으로 축적한다.
- 이미 있는 스킬과 중복 구현하는 낭비를 방지한다.

## 호출 원칙
- `/pre-feature-check` 또는 `/pre-feature-check <피처 키워드>`로 호출한다.
- 새 기능 작업 또는 `/plan` 직전에 실행한다.
- 키워드가 없으면 사용자에게 피처 주제를 먼저 물어본다.

## 입력
- 피처 키워드 (예: `spring boot security`, `pdf export`, `audit log`)
- 현재 프로젝트 경로의 스킬 목록

## 출력
- 현재 설치된 스킬 목록 (`.agents/skills/`, `.skill-lab/skills/`)
- skills.sh 검색 결과 (`npx skills find <키워드>`)
- 중복/보완 분석 결과
- 도입 제안 (설치 명령 + 이유) 또는 "기존 스킬로 충분" 판정

## 워크플로우

1. **현재 스킬 목록 확인**
   ```
   .agents/skills/ 디렉터리 나열
   .skill-lab/skills/ 디렉터리 나열
   ```

2. **키워드 추출**
   - 사용자가 제공한 피처명 또는 대화에서 주요 키워드 추출
   - 예: "로그인 기능 추가" → `authentication`, `login`, `jwt`

3. **skills.sh 검색**
   ```bash
   npx skills find <키워드1>
   npx skills find <키워드2>  # 필요 시
   ```

4. **비교 분석**
   - 검색된 스킬과 기존 스킬 목록 대조
   - 중복: 이미 동등한 기능이 있으면 "기존 스킬 활용 권장"
   - 보완: 기존에 없는 전문 지식이면 "도입 검토 대상"
   - 무관: 현재 피처와 관련 없으면 "패스"

5. **결과 출력**
   ```
   ## Pre-Feature Check: {피처명}

   ### 현재 스킬 ({N}개)
   - {스킬명}: {설명}
   ...

   ### skills.sh 검색 결과
   - {스킬명} ({설치수}) - {설명}
   ...

   ### 권장 액션
   - [도입] `npx skills add <패키지>` — {이유}
   - [패스] {스킬명} — {이유}
   - [기존 활용] {스킬명} — {이유}
   ```

## 게이트
- 피처 키워드가 1개 이상 있다.
- 현재 스킬 목록 확인이 완료됐다.

## 중단 조건
- 피처 키워드를 파악할 수 없다 → 사용자에게 피처 주제를 요청한다.

## 차단 시 반환 형식
- 현재 stage: `pre-feature-check`
- 차단 사유: 키워드 미제공
- 다음 필요 입력: "어떤 기능을 구현하려는지 알려주세요."
